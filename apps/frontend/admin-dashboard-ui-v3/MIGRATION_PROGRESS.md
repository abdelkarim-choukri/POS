## Notes / ground truth captured
- `vendorPaymentsApi` (lib/merchant/api.ts): `list` unwraps `{records}`→array; `outstanding(vendorId)` returns PO array directly; `summary(vendorId)` returns `{total_paid,total_outstanding,payment_count,avg_days_to_pay}`.
- Backend: `POST /business/vendor-payments` (owner/manager), `/confirm` & `/void` (**owner only**), `GET /vendors/:id/outstanding` & `/payment-summary`.
- `CreateVendorPaymentDto`: `amount_paid @IsNumber @Min(0.01)`, `payment_method @IsIn(['bank_transfer','cheque','cash','other'])`, `payment_date @IsNotEmpty`, optional `purchase_order_id @IsUUID`, `reference_number`, `notes`. `VoidVendorPaymentDto.reason @IsNotEmpty`.
- list/get return raw entity — **no** `vendor`/`purchaseOrder` relations; UI resolves names from vendors + PO lists.
